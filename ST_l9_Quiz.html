Software testing - Week 2 - lecture 9 - Quiz

1. What is the main objective of structural coverage criteria in software testing?





Structural coverage criteria focus on ensuring that structural elements like nodes, edges, and paths are adequately covered during testing.

2. Which of the following is NOT one of the main structural coverage criteria?





The main criteria are node, edge, edge-pair, and path coverage. Code branch optimization coverage is not one of them.

3. What does node coverage ensure in a control flow graph?





Node coverage ensures that every node or vertex in the control flow graph is executed at least once.

4. Which algorithm is commonly used to generate paths for node and edge coverage?





BFS and DFS are typically used to systematically explore all nodes and edges for generating test paths.

5. What does edge-pair coverage require?





Edge-pair coverage requires all possible sequences of two consecutive edges to appear in test paths.

6. Why is complete path coverage often infeasible in real programs?





Path coverage becomes infeasible when loops exist, as they can lead to infinitely many possible paths.

7. What is a prime path in graph-based testing?





A prime path is a maximal simple path that does not repeat nodes and is not a subpath of any other simple path.

8. What does the exclamation mark (!) indicate in the prime path finding algorithm?





Paths are marked with an exclamation mark when they reach nodes that have no outgoing edges.

9. What is the main purpose of generating test paths from prime paths?





Test paths are generated to extend prime paths from the start to end nodes, ensuring full executable paths.

10. What does round-trip coverage primarily test?





Round-trip coverage ensures that paths starting and ending at the same node are tested to verify cyclic program behaviors.

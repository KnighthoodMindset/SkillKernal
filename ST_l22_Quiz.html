Software testing - lecture 22 Quiz

What is a predicate in logic-based testing?





A predicate is any expression or formula that evaluates to true or false.

What are clauses in a predicate?





Clauses are the smallest Boolean components within a predicate that do not contain logical operators.

What does predicate coverage (PC) require?





Predicate coverage requires each predicate to evaluate to true and false at least once.

How does clause coverage (CC) differ from predicate coverage?





Clause coverage tests each clause individually, which may not make the entire predicate false, unlike predicate coverage.

What is combinatorial coverage (CoC) in logic-based testing?





CoC involves testing all combinations of truth values for clauses within a predicate.

Why is combinatorial coverage often impractical?





CoC is exhaustive, leading to an exponential number of test cases for predicates with many clauses.

What is the main idea of active clause coverage (ACC)?





ACC tests one major clause at a time, with minor clauses set to fixed values to observe the major clause's effect on the predicate.

Which ACC variation allows no restrictions on minor clause values?





GACC places no restrictions on the values of minor clauses while testing the major clause.

What does Correlated Active Clause Coverage (CACC) require?





CACC requires minor clauses to be set such that the major clause determines the predicate's outcome.

How does Restricted Active Clause Coverage (RACC) differ from CACC?





RACC requires minor clauses to keep the same values when the major clause is true and false, further constraining the tests.

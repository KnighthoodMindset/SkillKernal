Software testing - lecture 36 - Quiz

What is the primary purpose of mutation testing?





Mutation testing evaluates whether existing test cases can detect small syntactic or logical changes in a program.

In mutation testing, what is the term for the original version of the program?





The original program is called the ground string.

Which of the following describes a mutant?





A mutant is created by applying a mutation operator to the ground string.

What is a stillborn mutant?





A stillborn mutant cannot be compiled and is discarded.

What defines an equivalent mutant?





An equivalent mutant cannot be killed because it produces the same output as the original program.

What are the three key conditions in killing a mutant?





Reachability, infection, and propagation describe how a mutant affects program execution and output.

What is the difference between strong and weak killing of mutants?





Strong killing requires the output to differ, while weak killing only requires an internal state change.

Why are trivial mutants considered of little value in mutation testing?





Trivial mutants offer little insight because they are easily killed and do not assess test effectiveness.

In the minVal program example, which input condition can help kill the mutant where minVal = b replaces minVal = a?





Providing inputs where b > a ensures the output differs, killing the mutant.

Why is mutation testing considered computationally intensive?





Mutation testing is resource-intensive due to generating many mutants and designing meaningful test cases.

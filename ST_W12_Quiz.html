Software testing - Overall Quiz

State true or false: Regression testing is a white-box testing technique as it tests for the changes done to the code.



False. Regression testing is primarily a validation technique used to ensure that code changes do not introduce new faults; it is not limited to white-box testing.

Let P be a program and let P′ be a modified version of P. When do we say that a particular test case t is obsolete for P′?



A test case is considered obsolete if its input is no longer valid for the modified program P′, meaning it cannot be executed correctly or meaningfully on P′.

State true or false: Testing for non-functional or quality requirements can be done at all the phases of software development.



False. Non-functional testing such as performance, security, or usability is typically done during specific phases, often later in development or during system testing.

If a software application version X released in 2007 can be used on version Y of the same application released in 2010, what does it indicate regarding its compatibility?



Forward compatibility means that an older version of the software (X) can work with a newer version of the system or environment (Y).

If a particular file available in a web link is set to have “Readonly” access to all users, then it is an example of which kind of quality requirement?





Setting a file to “Readonly” for all users is a security authorization requirement, controlling the permitted actions on the resource.

Installation instructions are made available as a part of the software package. What are these instructions a part of?





Installation instructions are included as part of the software documentation to guide users in correctly installing and configuring the software.

Which of the following terms represents the testing that is done to find the upper limit capacity of the inputs that a particular system can handle?





Stress testing evaluates the upper limits of a system by pushing it beyond normal operational capacity to observe its behavior under extreme conditions.

Which of the following is a list of key challenges in testing of mobile applications?





Mobile testing challenges include handling device fragmentation, varying mobile networks, and ensuring compatibility across different platforms and devices.

When performance testing is considered for mobile applications, which of the following aspects are tested for?





Mobile performance testing focuses on network coverage, memory usage, and battery consumption to ensure the application performs well on real devices.

Which of the following is a list of interrupts that mobile applications are tested for?



Mobile applications are tested for interrupts like network outages and incoming or outgoing calls/messages to ensure they handle interruptions gracefully.

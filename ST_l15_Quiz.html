Software testing - lecture 15 - Quiz

What does a node represent in a control flow graph (CFG) for source code testing?





In a CFG, each node represents a statement or block of statements in the program.

What is the main purpose of extending a control flow graph with variable definitions and uses?





Adding definitions and uses to the CFG allows testers to apply data flow coverage criteria for thorough testing of variables.

What actions does a variable undergo in data flow testing?





In data flow testing, variables undergo definition (assignment) and use (reading or referencing in expressions or conditions).

What is a def-use path in data flow testing?





A def-use path is a sequence of statements connecting a variable’s definition to its use without redefinition in between.

Which data flow testing criterion ensures that each definition of a variable reaches at least one of its uses?





All-Defs Coverage ensures that each variable definition reaches at least one of its uses.

Which criterion in data flow testing is the strongest and requires every possible def-use path to be executed at least once?





All-DU-Paths Coverage is the most rigorous, requiring every possible definition-clear path from each definition to each use to be executed.

How does data flow testing help identify unused definitions and incorrect variable updates?





Data flow testing detects unused definitions and incorrect updates by ensuring that variable definitions correctly reach their uses along definition-clear paths.

Why is testing through graph models considered more structured?





Graph-based testing provides a structured view by analyzing the interactions of control flow and data flow in the program.

What are definition-clear paths in data flow testing?





Definition-clear paths are those connecting a variable’s definition to its use without any redefinition in between.

How does data flow testing strengthen structural testing of a program?





Data flow testing strengthens structural testing by analyzing both control flow and data flow, verifying variable lifetimes and dependencies.

Software testing - lecture 11 - Quiz

1. What is the main focus of data flow testing?





Data flow testing focuses on how data moves through a program and how variables are defined and used.

2. In data flow testing, when is a variable said to be defined?





A variable is defined when a value is assigned to it through an assignment, input, or parameter passing.

3. What does a control flow graph (CFG) represent in data flow testing?





A CFG represents the flow of control between statements or blocks in a program.

4. What is a definition–use (DU) pair?





A DU pair connects the location where a variable is defined to the location where it is used.

5. What is the main goal of testing based on DU pairs?





DU pair testing ensures that every defined variable is used and no variable is used without definition.

6. What is the difference between local use and global use of a variable?





Local use happens within the same statement, while global use occurs between different statements.

7. What is a computational use (c-use) of a variable?





A computational use occurs when a variable’s value is used in computations or output statements.

8. What is a predicate use (p-use)?





Predicate use occurs when a variable is used in a condition or decision-making statement.

9. What types of DU pairs can be derived from the classification of uses?





The two main types of DU pairs are definition–computation use (DCU) and definition–predicate use (DPU) pairs.

10. What is the primary purpose of applying data flow coverage criteria?





Data flow coverage ensures that all definition–use relationships are covered through proper test cases.

Software testing - lecture 40 - Quiz

What is the primary focus of the current part of the course?





The focus is on applying mutation testing for program integration testing.

Which programming paradigm is emphasized for object-oriented integration mutation testing in this course?





Object-oriented programming is emphasized, especially for languages like Java.

What is the purpose of the access modifier change operator?





It modifies access levels to test whether accessibility rules for variables and methods are enforced.

How does hiding variable deletion operator work in object-oriented integration testing?





Hiding variable deletion removes overriding or hidden variables so that references access parent or ancestor variables, detecting integration errors.

Which operator modifies the order or number of arguments in method calls?





These operators modify the order and quantity of arguments in method calls to test overloading usage.

What is the purpose of the overriding method deletion operator?





Overriding method deletion removes overriding methods so that references use the parent method, testing method invocation correctness.

Which operator tests correct usage of the this keyword?





The this keyword deletion operator tests if member variables are correctly accessed when hidden by local variables or parameters.

What does static modifier change test in object-oriented integration testing?





Static modifier change switches instance and class variables to test correct usage of static and instance members.

How does parent constructor deletion operator help in testing?





Parent constructor deletion removes calls to super constructors to verify whether default initialization of parent class leads to errors.

Approximately how many object-oriented integration mutation operators are described in this section?





There are about twenty mutation operators covering integration testing of methods, classes, inheritance, overloading, access levels, and related interactions.

Software testing - Overall Quiz

For the first five questions, consider the following set of requirements for a college job placement process. If the CGPA is ≥ 8 (out of 10), then the students are eligible for companies that offer salaries ≥ INR 20,00,000/= per annum. If the CGPA is between 6 and 8 then they are eligible for companies that offer salaries less than INR 20,00,000/= per annum. If there aren’t any companies with salary offers ≥ INR 20,00,000/= per annum in a particular period, then those with CGPA ≥ 8 will also appear in the companies in the lower salary categories. Those with less than 6 CGPA will not be assisted by the college placement office for their placements. Answer the following questions related to equivalence partitioning based testing of these requirements.

How many partitions will there be for the input CGPA if we consider only valid inputs?





Three partitions. CGPA is divided into three valid ranges: less than 6, 6 to less than 8, and 8 or above.

Do the test inputs {CGPA = 8, CGPA = 9, CGPA = 9.5} belong to the same partition?



Yes, all these values fall into the CGPA ≥ 8 partition.

What is the expected output for the test case containing CGPA = 8 as input?



The student is eligible for offers above INR 20,00,000, and if no such offers exist, they are also eligible for offers in the lower salary category.

A student has a CGPA of 6. Will they be eligible to appear for placements for companies offering a salary of INR 20,00,000 per annum?



False. A student with CGPA 6 is eligible only for companies offering salaries less than INR 20,00,000 per annum.

Which of the following is a correct partitioning of salary inputs?





Both options are valid partitions. Salary inputs can be divided in either way for equivalence partitioning testing.

Amongst the various coverage criteria for input space partitioning, which is the most expressive and which is the least expressive?





Each choice coverage is the least expressive, and all combinations coverage is the most expressive because it tests all possible input combinations.

State true or false: Multiple base choice coverage subsumes pair-wise coverage.



False. Multiple base choice coverage does not guarantee coverage of all pairwise combinations.

Which criterion below will have a maximum number of test cases?





All combinations criterion generates the maximum number of test cases because it considers all possible combinations of input partitions.

When does T-wise coverage criterion become the same as all combinations coverage criterion?



T-wise coverage becomes equivalent to all combinations coverage when T equals the number of input partitions.

State yes or no: While partitioning the inputs using equivalence partitioning, both valid and invalid inputs need to be considered.



Yes. Equivalence partitioning requires considering both valid and invalid inputs to ensure thorough testing.

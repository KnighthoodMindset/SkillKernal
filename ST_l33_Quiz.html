Software testing - lecture 33 Quiz

What is the main goal of Input Space Partitioning (ISP) in functional testing?





ISP divides input domains into partitions to systematically generate test cases and ensure meaningful coverage.

Which coverage criterion tests every possible combination of partitions from all input variables?





All Combinations Coverage (ACoC) is exhaustive and tests every possible combination of input partitions.

Why is Each Choice Coverage (ECC) considered weak?





ECC may miss critical interactions because it tests only one representative value from each partition.

How does Pairwise Coverage reduce the number of test cases compared to All Combinations Coverage?





Pairwise Coverage tests all pairs of partitions, covering important interactions without testing every full combination.

What does T-Wise Coverage represent?





T-Wise Coverage ensures that all combinations of T partitions are tested together.

How does Base Choice Coverage (BCC) select test cases?





BCC creates a base test using base choices and generates additional tests by varying one input at a time.

What is the key difference between BCC and MBCC?





MBCC extends BCC by allowing several base choices per input to increase coverage while controlling test case count.

In the triangle classification example, how many test cases are required for All Combinations Coverage if each side has 4 partitions?





Each of the 3 sides has 4 partitions, so 4 × 4 × 4 = 64 test cases for All Combinations Coverage.

What are infeasible test requirements in input space partitioning?





Infeasible requirements are partition combinations that are logically impossible in the program context.

How can infeasible test requirements be handled in Base Choice or Multiple Base Choice Coverage?





Strategic selection of base choices helps reduce infeasible test cases while maintaining meaningful coverage.

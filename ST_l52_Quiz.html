Software testing - lecture 52 - Quiz

What is the primary purpose of Dart in program testing?





Dart performs concolic execution, combining concrete and symbolic execution for systematic unit testing.

How does Dart explore neighboring paths after collecting initial path constraints?





Dart systematically explores neighboring paths by negating constraints from the stack and executing along these new paths.

What happens when Dart encounters a nonlinear constraint that cannot be handled by a solver?





Dart substitutes concrete values for variables in nonlinear constraints and sets a flag to indicate symbolic evaluation cannot continue.

During interface extraction in C programs, what types of inputs does Dart identify?





Dart identifies inputs from uninitialized memory, top-level function arguments, and values returned by external or library functions.

How does Dart initialize pointers in its random test driver for C programs?





Pointers are initialized randomly, using a coin toss to choose between null or newly allocated memory.

Which tool does Dart use to parse and analyze C programs for symbolic updates?





Dart uses the OCaml-based CIL tool to parse and analyze C programs for symbolic execution.

What are the possible termination outcomes for Dart?





Dart can terminate by successfully exploring all paths, detecting a bug with an abort, or potentially running indefinitely due to undecidable path exploration.

How does Dart handle external or library functions during test driver generation?





External or library functions are treated as unknown deterministic functions, substituting compatible type values without instrumenting their code.

Which tools have evolved from Dart for automated C program testing?





Dart has evolved into Cute for C programs, with related tools like Crest available for experimentation.

What is the role of the constraint solver in Dart?





The constraint solver is used to find input values that satisfy path constraints, allowing Dart to systematically explore different execution paths.

Software testing - lecture 31 Quiz

What is the main purpose of Equivalence Class Partitioning (ECP) in functional testing?





ECP divides the input domain into partitions so that any input in a partition is expected to produce similar program behavior.

What are the two essential properties that partitions must satisfy in Equivalence Class Partitioning?





Partitions must be disjoint (no overlap) and complete (cover all possible inputs).

In functional testing, what is considered the input domain of a program?





The input domain consists of all possible values that can be provided as inputs to the program.

What is an equivalence relation in the context of input space partitioning?





An equivalence relation groups inputs that are expected to produce the same behavior in a partition.

Which two major approaches are used for input domain modeling in ECP?





Interface-based and functionality-based approaches are the two main methods for defining partitions in the input domain.

How does the interface-based approach define partitions?





The interface-based approach partitions inputs based on individual parameters without considering interactions.

How does the functionality-based approach differ from the interface-based approach?





Functionality-based partitioning considers how inputs interact to affect overall program behavior.

Why must partitions include both valid and invalid values?





Including valid and invalid inputs ensures both normal behavior and error handling are tested.

What is the primary goal of equivalence class partitioning in terms of test case design?





ECP aims to select representative inputs to reduce the number of test cases while covering all relevant input scenarios.

Why is input space partitioning considered a critical component of black-box testing?





Input space partitioning systematically guides test case design by selecting representative inputs, essential for black-box functional testing.

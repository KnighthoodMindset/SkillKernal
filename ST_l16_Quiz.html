Software testing - lecture 16 - Quiz

What is the main purpose of integration testing in software development?





Integration testing focuses on verifying that combined modules interact correctly according to the software design.

Which of the following is NOT a type of interface used between software modules?





Procedure call, shared memory, and message passing are valid interface types; loop execution is not an interface type.

Why is testing interfaces critical in software systems?





Empirical studies indicate that about 25% of software errors originate from interface problems, making interface testing crucial.

What is the role of a test stub in integration testing?





A test stub simulates a missing lower-level module by returning fixed or simulated outputs for testing purposes.

How does a test driver function in integration testing?





A test driver simulates the higher-level module responsible for invoking lower-level modules during testing.

Which integration testing approach starts from the highest-level module and progresses downward?





Top-down integration testing begins with the highest-level module and uses stubs for lower-level modules initially.

In bottom-up integration testing, what is used to simulate unavailable higher-level modules?





Test drivers are used to simulate higher-level modules when they are not yet developed in bottom-up testing.

What is a key drawback of big bang integration testing?





Big bang testing integrates all modules at once, making it hard to isolate and identify the source of faults.

Which integration testing approach combines both top-down and bottom-up strategies?





Sandwich testing uses top-down testing for higher-level modules and bottom-up testing for lower-level modules before combining them.

How does integration testing strengthen overall software reliability?





Integration testing strengthens reliability by verifying that modules interact correctly and interfaces function properly, reducing potential errors.

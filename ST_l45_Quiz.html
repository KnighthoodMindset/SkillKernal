Software testing - lecture 45 - Quiz

What are the three broad categories of web application testing?





Web application testing is broadly categorized into static website testing, client-side testing, and server-side testing.

What is the main goal of system-level server-side testing in web applications?





System-level server testing aims to design test cases that reveal faults in dynamic web applications.

Which of the following can be considered severe errors in server-side testing?





Severe errors are those unhandled faults that may lead to permanent data corruption.

What does the Component Interaction Model (CIM) represent?





CIM models intra-component behavior where nodes correspond to atomic HTML sections.

What is an atomic section in the context of CIMs?





An atomic section is a portion of HTML sent entirely or not at all, possibly with dynamic content.

How does the Application Transition Graph (ATG) differ from a CIM?





ATG models inter-component behavior where each node represents a CIM of a server component.

Which transitions can occur within a CIM?





CIM transitions can include hyperlinks, form submissions, component outputs, abnormal client actions, and internal server redirects.

What are content variables in CIM and ATG models?





Content variables represent dynamic data such as user ID, password, or other session-dependent values.

Why is ATG creation largely manual compared to CIM generation?





ATG creation is mostly manual because it requires mapping inter-component server interactions, which are complex.

What are some active research challenges in server-side web application testing using CIMs and ATGs?





Research challenges include capturing system-level data flow, automating ATG generation, and handling concurrency for multiple clients.

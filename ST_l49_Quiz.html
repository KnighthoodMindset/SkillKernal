Software testing - lecture 49 - Quiz

What is the main challenge of symbolic execution in programs with loops?





Symbolic execution becomes challenging with loops because the number of possible execution paths can be infinite, depending on input values.

How does symbolic execution handle program inputs?





In symbolic execution, program inputs are treated as symbolic values rather than concrete data.

What do path constraints represent in symbolic execution?





Path constraints define the conditions under which certain paths are taken in the program.

Why can solving path constraints in symbolic execution be difficult?





Solving path constraints can be undecidable or computationally hard, particularly for predicates over integers or unknown functions.

What is concolic testing?





Concolic testing combines concrete and symbolic execution to systematically explore program paths.

How does concolic testing explore alternative program paths?





Concolic testing negates path constraints from symbolic execution and uses solvers to generate new inputs for alternative paths.

Why is concolic testing useful for programs with loops?





Concolic testing helps handle loops by systematically exploring feasible paths without getting stuck in infinite symbolic paths.

Name some tools used for symbolic and concolic testing.





Tools for symbolic and concolic testing include CUTE (C), JCUTE (Java), PEX (.NET), and SAGE for automated testing.

What is the main advantage of combining concrete execution with symbolic execution?





Concrete execution guides symbolic analysis, helping explore paths that would be hard to solve purely symbolically.

What practical technique has emerged to overcome scalability issues in symbolic execution?





Directed automated random testing (DART) uses concolic testing to overcome scalability and constraint-solving challenges.

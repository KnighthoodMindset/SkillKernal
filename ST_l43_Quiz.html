Software testing - lecture 43 - Quiz

What distinguishes web application testing from traditional program testing?





Web applications differ from traditional programs because they are deployed on the web and have distributed, heterogeneous, and dynamic characteristics.

Which of the following best describes a static website?





Static websites consist of fixed HTML files that remain the same for all users.

Why is testing dynamic web applications more complex than testing static websites?





Dynamic web applications are more complex because content is generated on demand, based on inputs, session data, and distributed components.

What is the primary criterion for testing static websites?





Edge coverage is used to ensure all hyperlinks function correctly in static websites.

Which layer of a web application is responsible for HTML and UI output?





The presentation layer handles HTML and user interface output in a web application.

What is a key challenge in web application testing due to the stateless nature of HTTP?





HTTP is stateless, so maintaining session state requires mechanisms like cookies or session objects.

Why are classical testing techniques limited for system-level web application testing?





Classical techniques like CFGs and call graphs are limited to unit testing and do not handle distributed, dynamic behavior.

In dynamic web applications, what does client-side testing primarily focus on?





Client-side testing focuses on script execution and UI behavior in the web browser.

Which factor complicates test case generation for dynamic web applications?





Dynamic inputs, session states, and runtime component changes make test case generation challenging.

Which of the following statements about web application testing is true?





Dynamic web application testing considers client-side scripts, server-side processing, and session state changes separately.

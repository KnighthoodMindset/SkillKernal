Software testing - lecture 44 - Quiz

What distinguishes web applications from traditional software in terms of testing?





Web applications are distributed, heterogeneous, and dynamic, requiring testing of multiple layers and integrated system-level tests.

Which layer of a web application is responsible for business logic and computations?





The data-content layer handles business logic and computations in a web application.

What is the main focus of client-side testing in dynamic web applications?





Client-side testing focuses on scripts, form controls, and browser behavior.

Why is exhaustive input testing often infeasible for realistic web applications?





Exhaustive input testing is infeasible due to the large and complex input space of web applications.

What is the purpose of bypass testing in web applications?





Bypass testing removes or alters client-side checks to see if the server safely handles invalid or malicious inputs.

Which of the following is NOT a typical client-side constraint that can be bypassed?





Server-side database schema is not a client-side constraint and cannot be bypassed from the browser.

Why are session-derived inputs often more effective than random inputs for web application testing?





Session-derived inputs are effective because they reflect realistic user interactions and patterns.

What challenge arises due to HTTP being stateless in web applications?





HTTP is stateless, so distributed state must be managed via cookies or session objects.

Which of the following should be included in client-side test suites for robustness and security testing?





Client-side test suites should include boundary values, malformed inputs, missing/extra fields, and altered request metadata.

Why must testers design inputs via the public interface rather than accessing server internals directly?





Testers must design inputs through the public interface because server internals are often inaccessible for security reasons.

Software testing - lecture 35 - Quiz

What is the main purpose of mutation testing?





Mutation testing evaluates whether existing test cases can detect small syntactic or logical changes in software artifacts.

In mutation testing, what is the original artifact before applying any mutation?





The original artifact before mutation is called a ground string.

What is a mutant in mutation testing?





A mutant is the result of applying a mutation operator to the ground string.

When is a mutant considered killed?





A mutant is killed if a test case detects a behavioral difference between the mutant and the original artifact.

How is the effectiveness of mutation testing measured?





Mutation effectiveness is measured using the mutation score, which is the percentage of killed mutants.

Which of the following defines how a valid syntactic element can be replaced in mutation testing?





A mutation operator specifies how a syntactic element is replaced to create a mutant.

Which mutation testing coverage criterion requires every mutant to be killed by at least one test case?





Mutation coverage ensures that each generated mutant is killed by at least one test case.

Why is exhaustive mutation testing often impractical?





Exhaustive mutation testing is impractical due to the enormous number of possible mutants that can be generated.

In mutation testing, how are invalid input scenarios tested?





Mutation operators can generate invalid input data to test program robustness against unexpected or incorrect inputs.

What is one major challenge of mutation testing despite its effectiveness?





Mutation testing is resource-intensive and requires expertise to choose operators and design test cases effectively.

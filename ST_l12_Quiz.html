Software testing - lecture 12 - Quiz

1. What is a DU-path in data flow testing?





A DU-path is a path that starts at a variable’s definition and ends at its use without redefinition along the path.

2. What is a definition path set (def-path set)?





A def-path set is the collection of all DU-paths starting from a specific variable definition.

3. What is a def-pair set?





A def-pair set contains all DU-paths from a definition to a specific use of a variable.

4. What does DU-tour mean in data flow testing?





A DU-tour traverses a definition–use subpath without redefining the variable along the way.

5. Which data flow coverage criterion ensures every definition of a variable reaches at least one use?





All-Defs Coverage ensures every definition of a variable reaches at least one of its uses.

6. How does All-Uses Coverage differ from All-Defs Coverage?





All-Uses Coverage requires that every definition reaches all its possible uses, making it stricter than All-Defs Coverage.

7. What is the most rigorous data flow coverage criterion?





All-DU-Paths Coverage is the most rigorous because it requires testing every possible path between definitions and uses.

8. Which hierarchy represents subsumption among data flow coverage criteria?





All-DU-Paths Coverage subsumes All-Uses Coverage, which subsumes All-Defs Coverage.

9. How is data flow coverage related to structural coverage criteria?





Since edges typically involve variable usage, All-Uses Coverage implicitly ensures edge coverage.

10. Which techniques are commonly used for data flow-based test data generation?





Common techniques include explicit search, random search, symbolic execution, and model checking.

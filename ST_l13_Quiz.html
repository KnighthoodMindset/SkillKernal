Software testing - lecture 13 - Quiz

What is a Control Flow Graph (CFG) in software testing?





A CFG represents program statements or blocks and the possible transfers of control between them.

What do nodes in a CFG represent?





Nodes in a CFG represent program statements or basic blocks that execute sequentially without internal branches.

How are edges in a CFG interpreted?





Edges represent possible transfers of control between nodes in a CFG.

How does an if-else statement appear in a CFG?





An if-else statement introduces a branching node with separate edges for each branch.

What special behavior does a break statement introduce in a loop's CFG?





A break statement exits the loop immediately, creating an edge to the statement after the loop.

How does a do-while loop differ from a while loop in CFG representation?





A do-while loop executes the body at least once before checking the predicate, forming a self-loop structure in the CFG.

How are try-catch blocks represented in a CFG?





In CFGs, the try block is sequential, and each catch block represents a possible alternate path for exception handling.

In the statistical program example, what nodes represent the loop bodies?





The loop body nodes perform computations such as sum, variance, and standard deviation.

How can edge coverage be achieved in a CFG?





Edge coverage is achieved by test paths that traverse every edge at least once in the CFG.

Why is modeling source code as a CFG useful for software testing?





CFGs allow systematic visualization of execution paths, enabling the application of coverage criteria for better testing.

Software testing - lecture 23 Quiz

What is the main purpose of logic coverage criteria in software testing?





Logic coverage criteria ensure that the behavior of predicates in a program is thoroughly tested.

What does predicate coverage require?





Predicate coverage tests the predicate to be both true and false at least once.

What is the difference between clause coverage (CC) and all combinations coverage?





Clause coverage tests each clause individually, while all combinations coverage tests every possible combination of clause truth values.

What is the focus of active clause coverage (ACC)?





Active clause coverage focuses on how each major clause influences the overall predicate.

Which ACC variation allows minor clauses to take any values?





GACC allows minor clauses to take any values while testing the major clause.

What is inactive clause coverage (ICC) used for?





ICC focuses on ensuring that the major clause does not affect the truth or falsity of a predicate.

How does restricted ICC differ from generalized ICC?





Restricted ICC requires minor clauses to assume the same values in pairs of test cases corresponding to the predicate being true and false.

Why might some logic coverage criteria be infeasible in practice?





Infeasibility arises when system constraints prevent certain combinations of clause values from being valid test cases.

Which coverage criterion subsumes predicate coverage?





All inactive clause coverage criteria subsume predicate coverage by testing predicates as both true and false.

What is the most comprehensive logic coverage criterion?





Combinatorial coverage tests the predicate for every possible combination of clause values, subsuming all other logic coverage criteria.

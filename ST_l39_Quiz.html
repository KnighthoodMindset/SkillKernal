Software testing - lecture 39 - Quiz

In which context is the primary focus of mutation testing for this course?





The course focuses on applying mutation testing to programs for integration testing using mutation operators at the interface level.

What is integration mutation in the context of mutation testing?





Integration mutation applies mutations at the interfaces between components rather than inside individual methods.

Which of the following is a key integration mutation operator?





Integration parameter variable replacement is a key operator that changes variables passed to called methods to compatible alternatives.

How does unary operator insertion work in integration mutation testing?





Unary operator insertion alters expressions in method calls by adding operators like negation to create different test scenarios.

What is the effect of method call deletion in integration mutation testing?





Method call deletion removes the call but maintains compilability by replacing the return value with a suitable constant if needed.

Why does integration testing become more complex in object-oriented languages?





Object-oriented features like encapsulation, inheritance, polymorphism, and overloading increase integration testing complexity.

What does encapsulation enforce in object-oriented programming?





Encapsulation enforces information hiding using access levels like private, protected, default, and public.

Which keyword allows explicit access to parent class members in inheritance?





The super keyword is used to explicitly access variables and methods from a parent class.

How does polymorphism affect attributes and methods?





Polymorphic attributes and methods can take on different types at runtime, allowing flexibility in object-oriented programs.

What is the difference between instance variables and class (static) variables?





Instance variables are specific to objects, while class variables (static) are shared among all instances.

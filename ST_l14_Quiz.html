Software testing - lecture 14 - Quiz

What is the purpose of using graph-based models like CFGs and DFGs in software testing?





Graph-based models like CFGs and DFGs are used to represent program execution flow and data dependencies, forming the basis of white-box testing.

How does statement coverage relate to node coverage in a control flow graph?





Statement coverage ensures that every executable statement is executed at least once, which corresponds to node coverage in the CFG.

What does branch coverage ensure in white-box testing, and which graph element does it correspond to?





Branch coverage ensures that every branch from decision points like if, switch, or loop conditions is executed, corresponding to edges in the CFG.

How does loop coverage relate to prime path coverage in graph-based testing?





Loop coverage executes loops under different conditions and aligns with prime path coverage, which tests all maximal simple paths including loops.

What is the difference between decision coverage and MCDC in white-box testing?





Decision coverage ensures each decision is executed at least once, while MCDC ensures each condition within a decision independently affects the outcome.

Define cyclomatic complexity and explain how it is calculated using the CFG.





Cyclomatic complexity measures the number of linearly independent paths in a CFG and is calculated as M = E - N + 2P, where E is edges, N is nodes, and P is connected components.

Why is cyclomatic complexity important in determining the number of test cases for branch coverage?





Cyclomatic complexity reflects the number of independent execution paths, which helps determine the minimum number of test cases required to achieve complete branch coverage.

What is basis path testing, and how does it use cyclomatic complexity to guide test case design?





Basis path testing uses cyclomatic complexity to determine all linearly independent paths in the CFG, designing a minimal set of test cases that cover all logical paths.

Explain the concept of a Decision-to-Decision (DD) path and its significance in CFG analysis.





A DD-path is a sequence of statements between two decision points in a CFG, used to analyze control flow between decisions and design finer-grained test cases.

What are maximal chains in a graph, and how do they relate to DD-path testing?





Maximal chains are paths whose internal vertices each have one incoming and one outgoing edge, and they form part of DD-paths used in testing decision-to-decision transitions.

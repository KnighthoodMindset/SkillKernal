Software testing - lecture 30 - Quiz

What is the main focus of functional (black box) testing?





Functional testing focuses on verifying software functionality based on inputs and expected outputs without considering internal implementation.

Which of the following is used to efficiently cover the input space in functional testing?





ECP or ISP divides the input domain into meaningful subsets, testing representative values from each class.

What does Boundary Value Analysis (BVA) focus on?





BVA tests inputs at and around the edges of equivalence classes, as most errors occur at boundary conditions.

Why is Boundary Value Analysis important alongside Equivalence Class Partitioning?





BVA complements ECP by verifying inputs at the boundaries, detecting off-by-one and edge errors.

What is the purpose of Decision Table Testing in functional testing?





Decision Table Testing lists conditions and corresponding actions to cover all combinations of inputs systematically.

How does Random Testing help in functional testing?





Random Testing generates inputs randomly to detect unexpected bugs and test robustness.

What is the main advantage of treating a program as a function in functional testing?





Treating a program as a function allows testing based solely on input-output relationships without internal knowledge.

In Input Space Partitioning, what is an equivalence class?





An equivalence class is a subset of inputs for which the program is expected to produce similar behavior.

Why are boundary values considered critical in testing?





Boundary values are critical since off-by-one errors and incorrect comparisons often occur at input boundaries.

How do functional testing techniques collectively improve software reliability?





Functional testing techniques together ensure coverage of typical and boundary inputs, systematically improving reliability.

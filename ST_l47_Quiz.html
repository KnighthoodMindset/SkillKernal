Software testing - lecture 47 - Quiz

What causes faults in object-oriented applications according to the passage?





Faults in object-oriented applications arise due to features such as inheritance, polymorphism, constructors, and visibility.

Which type of testing focuses on interactions among components in object-oriented programs?





Testing at intra-class and inter-class levels focuses on interactions among components in object-oriented programs.

What is a common fault called inconsistent type use?





Inconsistent type use occurs when a descendant class does not override inherited methods, so polymorphism is absent.

What is a state definition anomaly in object-oriented programs?





State definition anomaly arises when refining methods in a descendant class fail to define variables required by overridden methods.

What happens in a state definition inconsistency fault?





State definition inconsistency occurs when a local variable in a descendant class hides an inherited variable, leading to unexpected behavior.

What is a state visibility anomaly?





State visibility anomaly occurs when access levels prevent descendant methods from correctly using inherited variables.

Which fault occurs when overriding methods redefine the same variable inconsistently?





State defined incorrectly refers to overriding methods redefining the same variable inconsistently.

What is indirect inconsistent state definition?





Indirect inconsistent state definition occurs when extension methods in descendant classes define inherited variables, creating potential anomalies.

Why is static analysis challenging for object-oriented programs?





Dynamic behavior, polymorphism, overridden methods, and encapsulation make static analysis challenging in object-oriented programs.

What specialized approaches are needed for testing object-oriented programs?





Specialized approaches extend traditional testing to handle object-oriented coupling, polymorphism, and dynamic state interactions.

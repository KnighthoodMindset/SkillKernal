Software testing - lecture 46 - Quiz

What is the primary focus of testing object-oriented applications compared to procedural applications?





Object-oriented testing focuses on class-level interactions and relationships rather than just individual methods.

Which feature of object-oriented programming allows creating classes that encapsulate data and define connections between components?





Abstraction allows the creation of classes that encapsulate data and define connections between components.

What distinguishes subtype inheritance from subclass inheritance in object-oriented testing?





Subtype inheritance allows a subclass to be substituted for a parent class while maintaining expected behaviors, whereas subclass inheritance reuses methods without necessarily meeting parent specifications.

What defines a polymorphic method in object-oriented software?





Polymorphic methods are those where the actual method executed depends on the runtime type of the object.

Which level of object-oriented testing evaluates sequences of method calls within a single class?





Intra-class testing evaluates sequences of method calls within a single class.

What is the main challenge in determining method execution in object-oriented testing?





Polymorphism, inheritance, and dynamic binding make it hard to determine which method executes at runtime.

What are dataflow anomalies in object-oriented software?





Dataflow anomalies occur when overridden methods fail to define or propagate state variables needed by later method calls.

What is a yo-yo graph in object-oriented testing?





A yo-yo graph represents methods as nodes and calls (including overridden and inherited methods) as edges, capturing dynamic behavior.

How does a yo-yo graph help in object-oriented testing?





Yo-yo graphs illustrate method call sequences bouncing across class hierarchies, highlighting potential anomalies and faults.

Which feature primarily complicates object-oriented testing at both intra-class and inter-class levels?





Polymorphism, inheritance, and dynamic binding make object-oriented testing complex at intra-class and inter-class levels.

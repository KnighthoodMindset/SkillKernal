Software testing - lecture 32 Quiz

What is the main purpose of input space partitioning (ISP) in functional testing?





ISP divides input domains into partitions to systematically generate test cases and ensure meaningful coverage.

What is the difference between interface-based and functionality-based partitioning?





Interface-based focuses on individual inputs, while functionality-based considers interactions and overall program behavior.

Which coverage criterion requires testing every possible combination of input partitions?





ACoC is exhaustive, requiring all possible combinations of partitions to be tested.

Why is Each Choice Coverage (ECC) considered a weak coverage criterion?





ECC selects one value per partition, which may miss important interactions between inputs.

How does pairwise coverage reduce the number of test cases compared to All Combinations Coverage?





Pairwise coverage tests all pairs of input partitions, significantly reducing test cases while maintaining high fault detection.

What is T-Wise coverage in input space partitioning?





T-Wise coverage generalizes pairwise testing by ensuring all combinations of T input partitions are tested together.

How does Base Choice Coverage (BCC) select test cases?





BCC uses a base test with selected base choices and varies one input at a time to generate additional tests.

What advantage does Multiple Base Choice Coverage (MBCC) have over BCC?





MBCC selects multiple significant values as base choices for each input, testing them in combination to increase coverage.

Which coverage criterion subsumes all others due to its exhaustive nature?





ACoC is exhaustive and covers every possible combination, thus subsuming all other criteria.

Why are Pairwise or Base Choice coverage criteria often preferred in practice?





Pairwise and Base Choice criteria offer a practical balance between coverage and the number of test cases, making them efficient for real testing scenarios.

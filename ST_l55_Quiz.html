Software testing - lecture 55 - Quiz

What are the two broad categories of system-level testing?





System-level testing is categorized into functional testing, which validates specified requirements, and nonfunctional testing, which evaluates quality attributes.

What is the primary objective of functional testing at the system level?





Functional testing ensures that the software correctly implements the specified requirements.

Which standards provide metrics for nonfunctional system testing?





Nonfunctional testing evaluates the system against quality attributes defined by NISS metrics, often ending in “-ility” or “-ty.”

What does interoperability testing assess?





Interoperability testing ensures the system can work with other software, hardware, or platforms across OS, languages, and devices.

Which type of nonfunctional testing evaluates the system’s ability to function continuously over time?





Reliability testing measures the system’s ability to operate continuously and reliably over extended periods.

What is the purpose of scalability testing?





Scalability testing checks whether a system can handle increasing loads, such as concurrent users, while maintaining performance metrics.

Which performance testing type evaluates system limits beyond expected load?





Stress testing examines system behavior beyond expected loads to determine limits and potential failures.

Which tools are commonly used for performance testing?





Performance testing is supported by tools like HP LoadRunner, NeoLoad, JMeter, Rational Performance Tester, and Gatling.

What is the focus of regulatory testing?





Regulatory testing ensures the system adheres to domain-specific standards and safety-critical regulations like DO, IEC, MISRA, FSSAI, CE, or FCC.

Which aspects of nonfunctional system testing were not covered in this lecture?





The lecture did not cover regression testing, usability testing, graphical user interface testing, or detailed security and reliability testing.
